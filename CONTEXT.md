# Project Context (CONTEXT.md)

เอกสารนี้ใช้เพื่ออธิบายบริบทที่มาที่ไปและสถานะปัจจุบันของระบบ **Oracle Forms Repo** 

## 📌 ที่มาของโปรเจกต์
ระบบนี้ถูกสร้างขึ้นเพื่อตอบสนองความต้องการในการจัดการเวอร์ชันของไฟล์ (เน้น Oracle Forms) เนื่องจากในอดีตการส่งไฟล์ หรือการควบคุมเวอร์ชันทำได้ยากและมักจะมีปัญหาไฟล์ทับกัน การมีระบบรวมศูนย์กลางจึงช่วยแก้ปัญหานี้ได้

## 📊 สถานะระบบในปัจจุบัน (Current State)
1. **แยกส่วน (Decoupled Architecture):**
   - เดิมทีโปรเจกต์อาจจะรันรวมกัน แต่ปัจจุบันแยก Frontend ไปลง Vercel และ Backend ไปลง Render เรียบร้อยแล้ว เพื่อแก้ปัญหาข้อจำกัดของ Serverless Function บน Vercel เวลาอัปโหลดไฟล์ขนาดใหญ่
   
2. **การทำงานของ Storage:**
   - ได้มีการแก้ไขปัญหา **"ภาษาไทย"** ในชื่อไฟล์แล้ว โดยการบังคับให้ระบบแปลงชื่อไฟล์ทางกายภาพ (Physical File) เป็น `UUID` ตอนส่งไปบันทึกที่ Supabase Storage ส่วนหน้าเว็บและฐานข้อมูลยังคงดึงชื่อภาษาไทยไปแสดงผลได้ตามปกติ

3. **User Interface (UI):**
   - มีการปรับปรุง UI ให้มีความทันสมัย (Modern Minimalist)
   - ปัญหา Z-index ของ Modal ได้รับการแก้ไขด้วย React Portal แล้ว

## 🔗 สิ่งสำคัญที่ต้องรู้สำหรับผู้พัฒนา
- ระบบหลังบ้าน (Backend) ทำงานเป็น RESTful API
- ข้อมูลระบบ (System) และ ชนิดของไฟล์ (File Type) จะผูกกันแบบ Many-to-One
- ไฟล์ทุกไฟล์จะเชื่อมโยงกับ "เวอร์ชันล่าสุด" (`latest_version`) ในตาราง `files` และสามารถตรวจสอบประวัติได้จากตาราง `file_versions`
