# Testing Guide (TESTING.md)

แนวทางการทดสอบระบบสำหรับนักพัฒนา

## 🧪 1. การทดสอบด้วยตนเอง (Manual Testing)
### การทดสอบหน้าเว็บ (UI/UX)
- ลองอัปโหลดไฟล์ที่มีชื่อ **ภาษาไทย** สลับ **ภาษาอังกฤษ** ยาวๆ เพื่อตรวจสอบว่าหน้าเว็บแสดงผลผิดเพี้ยนหรือไม่
- ตรวจสอบ Modal ทุกตัว (หน้า Login, กดยืนยันลบไฟล์, กด Signout) ว่าต้องบังหน้าจอทั้งหมด 100% ไม่มีส่วนไหนของแถบเมนู (Header/Sidebar) ลอยทับ
- ตรวจสอบ Notification ว่าเด้งเตือนเมื่อมีการอัปโหลดไฟล์ในระบบอื่นหรือไม่

### การทดสอบ API (Postman / Insomnia)
- ยิง API `/api/files` โดยไม่แนบ Authorization Header ระบบต้องตอบกลับ HTTP Status `401 Unauthorized` เสมอ
- ลองยิง API ลบไฟล์ข้ามสิทธิ์ หรืออัปโหลดไฟล์ที่ไม่ได้รับอนุญาต

## 🛠 2. การทดสอบอัตโนมัติ (Automated Testing)
*หมายเหตุ: ปัจจุบันระบบยังไม่ได้ขึ้นโครง Automated Test อย่างเต็มรูปแบบ (อยู่ในแผนงานถัดไป)*

**แต่คุณสามารถทดสอบโครงสร้าง TypeScript เบื้องต้นได้โดยรัน:**
```bash
# ฝั่ง Backend
cd backend
npm run build

# ฝั่ง Frontend
cd frontend
npm run build
```
หากมีข้อผิดพลาดของ Typescript (`TSxxxx`) ระบบจะแจ้งเตือนและยกเลิกกระบวนการ Build ทันที ถือเป็นการทำ Static Analysis ในตัว
