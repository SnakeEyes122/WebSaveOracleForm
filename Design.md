# Design Document (Design.md)

เอกสารนี้รวบรวมแนวคิดการออกแบบ UI/UX และ Design System ของระบบ Oracle Forms Repo

## 🎨 1. แนวคิดการออกแบบ (Design Concept)
- **Modern & Minimalist:** เน้นความเรียบง่าย สะอาดตา และลดทอนองค์ประกอบที่ไม่จำเป็น เพื่อให้ผู้ใช้โฟกัสที่ "ไฟล์" และ "ข้อมูล" เป็นหลัก
- **Responsive & Fluid:** รองรับการใช้งานผ่านหน้าจอที่หลากหลาย แม้ปัจจุบันจะเน้นการใช้งานบน Desktop เป็นหลัก

## 🌗 2. ธีมและโทนสี (Theme & Color Palette)
ระบบรองรับทั้ง **Light Mode** และ **Dark Mode** อย่างสมบูรณ์แบบ
- **Backgrounds:** ใช้สีขาว (White) หรือเทาอ่อนมากสำหรับ Light Mode และสีดำ/เทาเข้ม (`#0a0a0a`, `gray-900`) สำหรับ Dark Mode
- **Borders:** เส้นขอบใช้สีเทาอ่อน (`gray-200`) ในโหมดปกติ และเทาเข้ม (`gray-800`) ในโหมดมืด เพื่อสร้างกรอบที่ดูไม่แข็งจนเกินไป
- **Primary Color:** ใช้สี **ดำ/ขาวสลับกัน** (Monochrome) สำหรับปุ่มหลัก (เช่น ปุ่ม Upload, Add User) เพื่อความพรีเมียม
- **Accent Colors:** 
  - **สีแดง (Red):** ใช้สำหรับการกระทำที่ต้องระวัง (Destructive Actions) เช่น ปุ่มลบไฟล์, ลบผู้ใช้งาน (`red-500` ถึง `red-600`)
  - **สีน้ำเงิน (Blue):** ใช้สำหรับการแจ้งเตือน (Notifications) หรือสิ่งที่กำลัง Active

## 🔤 3. ตัวอักษร (Typography)
- **Sans-serif:** ใช้เป็นค่าเริ่มต้นสำหรับ UI ทั่วไป (เช่น ชื่อเมนู, ชื่อหน้าต่าง) เพื่อความอ่านง่าย
- **Monospace (Mono):** ถูกนำมาใช้อย่างจงใจสำหรับข้อมูลเชิงเทคนิค เช่น ชื่อไฟล์, เวอร์ชัน, หรือวันที่และเวลา เพื่อให้ข้อมูลดูเป็นระเบียบและให้ความรู้สึกเหมือนระบบสำหรับนักพัฒนา (Developer-focused)

## 🧩 4. องค์ประกอบ UI (UI Components)
- **Icons:** ใช้ `lucide-react` ทั้งระบบ เพื่อให้ได้ไอคอนแบบเส้นที่บางและดูทันสมัย
- **Modals / Popups:** บังคับใช้ `createPortal` เสมอ เพื่อนำ Modal ออกไปเรนเดอร์ที่ระดับ `body` แก้ปัญหา Z-Index ทับซ้อน และมี Backdrop เบลอพื้นหลังเล็กน้อย (`backdrop-blur-sm`)
- **Tables:** ตารางออกแบบให้กว้างเต็มขอบ (Full-width) ไม่มีเส้นขอบแนวตั้งกั้นระหว่างคอลัมน์ เพื่อให้ดูโปร่งและทันสมัย
