# Product Document (Product.md)

เอกสารข้อมูลผลิตภัณฑ์ (Product Requirements Document - PRD) สำหรับระบบ Oracle Forms Repo

## 🚀 1. วิสัยทัศน์ของผลิตภัณฑ์ (Product Vision)
เป็นแพลตฟอร์มศูนย์กลาง (Centralized Platform) ที่ใช้สำหรับจัดเก็บ ควบคุมเวอร์ชัน และแจกจ่ายไฟล์เฉพาะทาง (โดยเฉพาะไฟล์สกุล `.fmb`, `.fmx`, `.rdf` ของ Oracle) ให้แก่นักพัฒนาและผู้เกี่ยวข้องในองค์กร เพื่อลดปัญหาความซ้ำซ้อนของไฟล์และป้องกันไฟล์สูญหาย

## 👥 2. กลุ่มเป้าหมายและสิทธิ์ผู้ใช้งาน (User Personas & Roles)
ระบบออกแบบมาเพื่อรองรับผู้ใช้งาน 3 กลุ่มหลัก:
1. **Admin (ผู้ดูแลระบบ):** 
   - จัดการข้อมูลพื้นฐานได้ทั้งหมด (เพิ่ม/ลบ ระบบงาน, นามสกุลไฟล์, ผู้ใช้งาน)
   - มีสิทธิ์ลบไฟล์ในระบบ และดู Audit Logs ประวัติการเข้าใช้งานได้
2. **User (นักพัฒนา / Developer):**
   - มีหน้าที่อัปโหลดไฟล์ หรือส่งมอบงาน (Deployment) เข้าสู่ระบบ
   - สามารถแก้ไข อัปโหลดทับไฟล์เดิมเพื่อสร้างเวอร์ชันใหม่ได้ แต่ไม่สามารถลบไฟล์ทิ้งถาวรได้
3. **Viewer (ผู้รับมอบงาน / ผู้ตรวจ):**
   - สามารถเข้ามาในระบบเพื่อกด **ดาวน์โหลด** ไฟล์เวอร์ชันล่าสุดไปใช้งาน หรือดูประวัติการอัปเดตไฟล์ได้เท่านั้น
   - จะไม่เห็นปุ่มสำหรับจัดการข้อมูลใดๆ

## 🌟 3. ฟีเจอร์หลัก (Core Features)
- **Smart Versioning (ระบบจัดการเวอร์ชันอัตโนมัติ):** เมื่อผู้ใช้อัปโหลดไฟล์ที่มีชื่อตรงกับไฟล์เดิมในระบบ (ผูกกับระบบงานเดียวกัน) ระบบจะทำการปรับเลขเวอร์ชันขึ้นให้อัตโนมัติ (เช่น `1.0` ➡️ `1.1`) และเก็บประวัติไฟล์เก่าไว้
- **Robust Storage (ระบบจัดการไฟล์ทางกายภาพ):** หน้าเว็บจะแสดงชื่อไฟล์ดั้งเดิม (เช่น `ระบบบัญชี.fmx`) แต่หลังบ้านจะถูกจัดเก็บด้วยรหัส UUID สุ่ม เพื่อป้องกันปัญหาเรื่องภาษาไทยและการถูกแฮ็กผ่าน Path
- **Real-time Notifications:** มีกระดิ่งแจ้งเตือนเมื่อมีการอัปโหลดไฟล์ใหม่ เพื่อให้ผู้ที่เกี่ยวข้องทราบทันที (ทำงานผ่านระบบ Polling ทุกๆ 30 วินาที)
- **Comprehensive Audit Trail:** ทุกการกระทำสำคัญ เช่น การล็อกอิน การอัปโหลด การลบ จะถูกบันทึกไว้เสมอ เพื่อความโปร่งใสและตรวจสอบได้ (Accountability)

## 📈 4. ตัวชี้วัดความสำเร็จ (Success Metrics)
- ลดระยะเวลาในการค้นหาและส่งมอบไฟล์ระหว่างนักพัฒนา
- ลดอุบัติเหตุการนำไฟล์เก่า (Outdated) ไปใช้งาน
- ระบบต้องสามารถจัดการไฟล์ขนาดใหญ่ได้โดยไม่มีปัญหา Timeout (แก้ไขแล้วโดยการแยก Backend ไปยัง Render)
